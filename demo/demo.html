Step-by-Step Testing Flow

  1. Page View Stored: As soon as this page loads, a page_view event is generated and stored in the offline queue (visible in `localStorage`).
  2. Click a Button: Click the "Primary Action" button above. A click event with your exact (x,y) coordinates will be captured.
  3. Network Call: The SDK automatically batches events. After a short delay (3 seconds), you can look at the Network tab in Developer Tools to see a POST request sent to http://localhost:5000/api/events.
  4. Backend Processing: The Express server receives the batch and stores the documents in your MongoDB Atlas cluster.
  5. Dashboard Update: Visit your running frontend dashboard (`http://localhost:5173`). The new session, page view, and click coordinates will automatically appear in the metrics and heatmap!

Data Verification Steps

To verify the data in MongoDB Atlas:

  1. Log into your MongoDB Atlas dashboard.
  2. Click on your cluster name (`Cluster0`), then click Browse Collections.
  3. Expand the insightflow database and select the events collection.
  4. You will see documents that look exactly like this:
{
  "sessionId": "abc12345-uuid-format",
  "eventType": "click",
  "pageUrl": "/demo.html",
  "coordinates": {
    "x": 234,
    "y": 451
  },
  "timestamp": "2026-06-22T08:00:00.000Z"
}
